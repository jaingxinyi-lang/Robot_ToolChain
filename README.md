# 布米调试工具（Robot_ToolChain）

## 1. 概述

布米调试工具是一款基于 Qt5 的桌面应用，面向机器人调试与标定工程师，目标是在受信任的内网环境下提供脚本配置、远端执行、实时日志、脚本包部署和测试结果展示能力。

项目使用 Qt5 + C++17 + CMake 开发，通信方式支持 SSH 与串口二选一。同一时间只能有一种通信方式处于连接和执行状态。

## 2. 主要功能

- 通信配置
  - 菜单“设置 → 通信配置...”中选择 SSH 或串口。
  - 默认通信方式为串口，默认波特率为 `1500000`，波特率支持手动输入。
  - SSH 参数包括 host、用户名、端口、密码。
  - 串口参数包括端口、波特率、数据位、校验、停止位、流控和命令超时。
  - 串口模式假设开发板已经进入 Linux shell 控制台，可直接接收命令并输出日志。

- 脚本配置
  - 多槽位脚本配置，包括名称、远端路径、附加参数与超时。
  - 路径浏览会复用当前已连接的通信方式执行远端 `ls` 命令。

- 脚本执行
  - 顶部“一键配置”按顺序执行所有已配置脚本。
  - 脚本面板支持单独执行任意槽位脚本。
  - SSH 模式使用 libssh channel 执行远端 shell 命令。
  - 串口模式通过串口 shell 发送命令并实时接收输出。

- 脚本包部署
  - 顶部“部署脚本包”按钮在通信连接成功后可用。
  - 部署过程中会禁用“一键配置”和脚本按钮。
  - 程序从可执行文件同级或上级开发目录查找 `zip/JIAOBEN.zip`。
  - SSH 模式通过 SFTP 上传到开发板，再解压到 `/JIAOBEN`。
  - 串口模式通过 base64 分片上传，再在开发板解码并解压到 `/JIAOBEN`。
  - 部署成功后，程序会读取 `mapping/test_path.csv` 的“测试脚本”列，并自动把脚本配置为 `/JIAOBEN/<测试脚本>`。
  - 开发板需要提供 `unzip`；串口部署还需要提供 `base64`。

- 执行结果显示
  - 底部“执行结果”区域会根据映射表生成测试结果卡片。
  - 默认状态为蓝灯，脚本输出成功码 `SUCXXX` 时变绿，输出错误码 `ERRXXX` 时变红。
  - 映射表优先使用 CSV，建议放在程序同级目录或 `mapping/` 目录。

## 3. 映射表 CSV 格式

CSV 路径固定为可执行文件同级目录下的 `mapping/test_path.csv`。CSV 至少包含四列：测试项、测试脚本、成功码、错误码。第一行可以是标题行。

```csv
测试项,测试脚本,成功码,错误码
压力测试,test_stress.sh,SUC100,ERR100
摄像头,camera_opencv.sh,SUC200,ERR200
```

## 4. 使用说明

1. 打开“通信配置...”，选择 SSH 或串口并保存参数。
2. 点击顶部连接按钮建立通信连接。
3. 如需先部署脚本包，点击“部署脚本包”。
4. 打开“脚本配置...”配置远端脚本路径和参数。
5. 点击“一键配置”或打开脚本面板执行单个脚本。
6. 在 `terminal_text` 中查看传输、解压和脚本输出过程；在“执行结果”区域查看各测试项目状态。

## 5. 注意事项

- SSH 和串口不会同时运行。修改通信方式前需要先断开当前连接。
- 部署到 `/JIAOBEN` 需要当前开发板用户具备根目录写权限。
- 串口传输 ZIP 的速度受波特率影响，文件较大时耗时会明显增加。
- 密码以明文形式保存在本地设置中，仅适用于受信任的内网调试环境。
